# Public Release Checklist

Use this checklist before pushing the repository to a public remote or sharing
it with reviewers.

## Required Checks

Run the publication audit from the repository root:

```bash
python tools/sanitize_for_publication_git.py
```

The script fails on critical issues such as private keys, local machine paths,
credential-like files, or unresolved secret markers.

## What Should Be Public

This repository is intended to contain:

- `pbc/`: reference implementation.
- `examples/`: reproducible demonstrations and evaluation scripts.
- `results/`: compact experiment summaries used by the paper and README.
- `paper/`: public paper source, public PDF, and figures.
- `web/`: browser verifier demo code.
- `README.md`, `LICENSE`, and `setup.py`.

## What Must Not Be Public

Do not add or commit:

- Private keys or signing material: `*.key`, `*.pem`, `*.p12`, `*.pfx`.
- Local TLS certificates: `web/server.key`, `web/server.crt`.
- Local environment files: `.env`, `.env.*`.
- Patent drafts, cover sheets, or non-public legal documents.
- Private review notes, private email drafts, or named-recipient outreach plans.
- Local absolute paths such as user-profile folders, cloud-sync workspace
  folders, or drive-specific development paths.

## Generated Artifacts

The `output/` directory is generated by examples and is ignored for future
commits. Prefer keeping only curated figures and compact result text files in
the public history. If an output image is needed for the README or paper, move
it into `paper/figures/` with a stable filename and document how to regenerate
it.

## Personal Information

The repository intentionally includes author attribution and contact details in
the paper, README, and package metadata. If public email exposure is not wanted,
replace those values before publication.

## Image Rights

Before publishing, confirm that `examples/img/leo.jpg` and all derived figures
may be shared publicly. If there is any doubt, replace the image with a clearly
licensed sample and regenerate the example outputs.

## Reviewer Reproducibility

For review, keep the commands in the README working from a fresh clone:

```bash
pip install numpy Pillow
python examples/demo.py
python examples/format_robustness.py
python examples/forest_scatter_test.py
python examples/multi_image_eval.py
```

Some optional examples require additional dependencies; keep those documented in
the README rather than committing generated local files.
